# 《肉肉守摊大战僵尸》最小 MVP 游戏计划书

## 1. 项目概述

项目名称：肉肉守摊大战僵尸

游戏类型：小型横版格子塔防小游戏

核心主题：夜市烧烤摊被僵尸袭击，玩家需要摆放各种“肉肉守卫”，保护烤炉和摊主。

参考方向：借鉴经典格子塔防的基础体验，但主题、资源、单位和节奏采用夜市烧烤创意，避免完全照搬。

MVP 目标：做出一个可以完整游玩的最小版本，包含开始游戏、放置单位、资源增长、僵尸进攻、攻击防守、胜利失败和重新开始。

## 2. 核心玩法

玩家在 3 行 x 7 列的战场中放置肉肉单位。僵尸从右侧出现并向左移动。肉肉单位自动攻击或阻挡僵尸。玩家需要合理使用资源，坚持到倒计时结束。

胜利条件：坚持 90 秒，并且没有僵尸突破烤炉防线。

失败条件：任意僵尸到达最左侧烤炉线。

单局时长：约 90 秒。

## 3. 游戏地图

地图尺寸：3 行 x 7 列。

可放置区域：前 6 列。

僵尸出生区域：最右侧。

地图示意：

```text
烤炉 | 格子1 | 格子2 | 格子3 | 格子4 | 格子5 | 格子6 | 僵尸入口
烤炉 | 格子1 | 格子2 | 格子3 | 格子4 | 格子5 | 格子6 | 僵尸入口
烤炉 | 格子1 | 格子2 | 格子3 | 格子4 | 格子5 | 格子6 | 僵尸入口
```

## 4. 资源系统

资源名称：炭火值。

资源用途：召唤肉肉单位。

资源规则：

- 初始炭火值：100。
- 每 3 秒自动增加 25 炭火值。
- MVP 阶段暂不设计额外产能单位。
- 资源上限可设置为 999，也可以暂时不设上限。

## 5. 玩家单位

MVP 只制作 3 个玩家单位，保证开发范围可控。

| 单位名称 | 费用 | 血量 | 攻击 | 攻速 | 射程 | 定位 |
|---|---:|---:|---:|---:|---|---|
| 羊肉串射手 | 100 | 100 | 20 | 1.2 秒/次 | 同行全屏 | 基础远程输出 |
| 五花肉墙 | 75 | 400 | 0 | 无 | 无 | 高血量阻挡 |
| 爆汁丸子 | 125 | 1 | 120 | 一次性 | 当前格及前后一格 | 范围爆炸 |

### 5.1 羊肉串射手

功能说明：

- 放置后自动检测同一行是否有僵尸。
- 如果同一行存在僵尸，则发射竹签。
- 竹签直线向右飞行。
- 竹签命中第一个僵尸后造成伤害并消失。

### 5.2 五花肉墙

功能说明：

- 不主动攻击。
- 主要作用是阻挡僵尸前进。
- 僵尸接触后停止移动并开始啃咬。
- 血量归零后消失。

### 5.3 爆汁丸子

功能说明：

- 放置后等待 1 秒。
- 爆炸后对当前格子、前一格、后一格范围内的僵尸造成伤害。
- 爆炸后自身消失。
- 适合作为紧急清场单位。

## 6. 僵尸设计

MVP 只制作 3 种僵尸。

| 僵尸名称 | 血量 | 移速 | 攻击 | 攻速 | 特点 |
|---|---:|---|---:|---:|---|
| 普通饿僵尸 | 100 | 普通 | 20 | 1 秒/次 | 基础敌人 |
| 锅盖僵尸 | 220 | 慢 | 20 | 1 秒/次 | 血量较高 |
| 外卖僵尸 | 70 | 快 | 15 | 0.8 秒/次 | 移动快但血少 |

### 6.1 普通饿僵尸

最基础的敌人，用于建立游戏节奏和测试核心战斗。

### 6.2 锅盖僵尸

血量较高，移动较慢，用于测试玩家输出是否足够。

### 6.3 外卖僵尸

移动速度较快，血量较低，用于制造防线压力。

## 7. 关卡设计

MVP 只制作 1 个完整关卡。

关卡时长：90 秒。

敌人节奏：

| 时间段 | 进攻内容 |
|---|---|
| 0-15 秒 | 少量普通饿僵尸 |
| 15-35 秒 | 普通饿僵尸数量增加 |
| 35-60 秒 | 加入外卖僵尸 |
| 60-80 秒 | 加入锅盖僵尸 |
| 80-90 秒 | 最后一波混合进攻 |

示例刷怪表：

| 出现时间 | 僵尸组合 |
|---:|---|
| 5 秒 | 普通饿僵尸 x1 |
| 12 秒 | 普通饿僵尸 x1 |
| 20 秒 | 普通饿僵尸 x2 |
| 32 秒 | 外卖僵尸 x1 |
| 42 秒 | 普通饿僵尸 x2，外卖僵尸 x1 |
| 55 秒 | 锅盖僵尸 x1 |
| 68 秒 | 普通饿僵尸 x2，锅盖僵尸 x1 |
| 82 秒 | 外卖僵尸 x2，锅盖僵尸 x1 |

## 8. 玩家操作

基础操作：

1. 点击上方肉肉卡片。
2. 鼠标移动到战场格子。
3. 点击空格子放置单位。
4. 如果炭火值不足，则无法放置。
5. 如果格子已经有单位，则无法重复放置。

可选快捷键：

```text
1 = 羊肉串射手
2 = 五花肉墙
3 = 爆汁丸子
```

## 9. 界面需求

### 9.1 主界面

- 游戏标题。
- 开始游戏按钮。
- 夜市烧烤摊背景。

### 9.2 游戏界面

- 炭火值显示。
- 倒计时显示。
- 3 个肉肉单位卡片。
- 3 x 7 战场格子。
- 烤炉防线提示。
- 暂停按钮，可选。

### 9.3 结束界面

- 胜利弹窗。
- 失败弹窗。
- 重新开始按钮。
- 返回主界面按钮。

## 10. 美术风格

整体风格：轻松、卡通、夜市、烧烤摊。

视觉关键词：

- 暖色灯光。
- 炭火烤炉。
- 烧烤摊招牌。
- 木桌和地摊元素。
- 可爱搞笑的僵尸。

MVP 可以先使用简单占位图：

- 羊肉串射手：一串会发射竹签的烤串。
- 五花肉墙：厚厚一块五花肉。
- 爆汁丸子：圆滚滚的肉丸。
- 普通僵尸：绿色基础僵尸。
- 锅盖僵尸：头顶锅盖或拿锅盖。
- 外卖僵尸：背着外卖箱。

## 11. 音效需求

MVP 最少音效：

- 按钮点击。
- 放置肉肉。
- 竹签发射。
- 命中僵尸。
- 僵尸啃咬。
- 爆汁丸子爆炸。
- 胜利音效。
- 失败音效。

背景音乐优先级较低，可以后续补充。

## 12. 核心系统拆分

建议按以下模块开发：

| 模块 | 职责 |
|---|---|
| GameManager | 游戏状态、倒计时、胜负判定 |
| GridManager | 格子生成、格子坐标、单位放置、占用状态 |
| ResourceManager | 炭火值增长、消耗、显示 |
| UnitManager | 玩家单位生成和行为管理 |
| EnemyManager | 僵尸生成、波次控制 |
| ProjectileSystem | 竹签飞行、碰撞、命中伤害 |
| UIManager | 按钮、卡片、数值、弹窗 |
| AudioManager | 音效播放 |

## 13. MVP 开发顺序

推荐开发顺序：

1. 创建 3 x 7 格子地图。
2. 实现点击卡片并放置单位。
3. 实现炭火值增长和消耗。
4. 实现僵尸从右往左移动。
5. 实现僵尸遇到单位后停止并攻击。
6. 实现羊肉串射手自动攻击。
7. 实现竹签飞行、命中和伤害。
8. 实现五花肉墙阻挡。
9. 实现爆汁丸子延迟爆炸。
10. 实现敌人波次刷新。
11. 实现胜利和失败判定。
12. 添加基础 UI。
13. 添加基础音效和美术占位。
14. 调整数值，让一局可以正常完成。

## 14. 最小完成标准

MVP 完成时必须满足：

- 可以进入游戏。
- 可以获得炭火值。
- 可以放置 3 种肉肉单位。
- 僵尸会按时间出现。
- 僵尸会移动、攻击和死亡。
- 玩家单位会攻击或阻挡。
- 有明确胜利和失败。
- 游戏结束后可以重新开始。

## 15. 暂不制作内容

为了控制 MVP 范围，以下内容暂不制作：

- 多个关卡。
- Boss 战。
- 肉肉升级系统。
- 大量技能和道具。
- 复杂剧情。
- 复杂经济系统。
- 账号、存档、排行榜。

## 16. 后续扩展方向

MVP 完成后可以逐步增加：

- 新肉肉：烤鸡翅、肥牛卷、烤肠炮台、辣椒牛肉。
- 新僵尸：厨师僵尸、冰箱僵尸、大胃王僵尸。
- 主动技能：孜然粉加攻、辣椒粉范围伤害、冰饮全屏减速。
- 更多夜市场景。
- 多关卡推进。
- 简单养成和单位升级。

## 17. 一句话总结

《肉肉守摊大战僵尸》是一个以夜市烧烤摊为主题的小型格子塔防游戏。MVP 阶段只做 3 个肉肉单位、3 种僵尸、1 个 90 秒关卡，目标是先完成一局能玩、有胜负、有基本策略的核心体验。
